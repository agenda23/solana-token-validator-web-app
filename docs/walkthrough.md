# 🚀 Solana Token Validator 開発ウォークスルー

本ドキュメントでは、Solana Token Validator の開発プロセスと、主要な機能の設計思想について解説します。

## 🛠 開発フェーズ

### 1. 基盤構築 (CA抽出とAPI連携)
- **正規表現によるCA抽出**: ユーザーが入力した長いURLや文字列から、Solanaのアドレス（ベース58形式、32~44文字）を正確に取り出すロジックを実装。
- **マルチソース解析**: 
    - **DEXScreener**: 価格、流動性、時価総額（FDV）、Boost状況、公式リンク（Web/SNS）。
    - **GoPlus Security**: ハニーポット判定、Mint/Freeze/Burn状況。

### 2. グレーディング・ロジックの進化 (S~F-)
単なる100点満点のスコア表示から、より実戦的な「7段階プロ・グレーディング」へ進化させました。
- **Bランク (Barebones) の導入**: 「技術的には安全だが、Websiteも何も設定されていない」という、開発者が投げ出したか早期すぎるトークンを隔離。
- **Sランクの厳格化**: セキュリティが100点でも、コミュニティリンクとBoost広告がない限りSランクは付与されない仕様。
- **F- (Deadly)**: ハニーポット（売却不可）は他のスコアを無視して最下位に固定。

### 3. UI/UX の洗練
プロツールとしての使い勝手を追求しました。
- **ハイブリッド数値表示**: `$3.9M` という直感的な短縮表記と、`$3,952,642` という信頼性を担保する全桁表示を同時に実現。
- **右側垂直パネル**: 最も重要な「スコア」「価格変化」「取引数」をカードの右側にまとめ、視線の動きを最適化。
- **カスタムCSSの適用**: 透過背景の `metric-card` を採用し、情報が詰まっていても圧迫感のないデザインへ。
- **履歴管理**: 解析するたびにカードが上に積まれていき、Xボタンで個別に削除可能な履歴機能を実装。

### 4. エラーハンドリングと堅牢性
- **DEXScreener未掲載への対応**: 上場直後のトークンでも、GoPlusのセキュリティチェックだけは先行して実行できるよう、APIの欠損を許容するフェイルセーフを設計。
- **型チェックの徹底**: APIレスポンスが `None` の場合でも属性エラーにならず、安全にデフォルト値を返すように実装。

## 💡 今後の展望
- **価格チャート（TradingView）の埋め込み**: カード内にミニチャートを表示。
- **主要ホルダー分析**: トップホルダーの保有率による中央集権性リスクの可視化。
- **リアルタイム通知**: お気に入りのCAのステータス変化をキャッチする機能。
